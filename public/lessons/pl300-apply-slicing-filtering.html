Apply Slicing and Filtering

Slicers and filters enable users to explore data dynamically. Slicers are visual controls, filters constrain data at query-level. Together they transform static reports into interactive analysis tools.

ElementTypePurpose

Key Principle
Slicers empower users. Filters constrain data. Use both strategically to guide analysis without limiting exploration.

Slicers

🎚️ Slicer Basics▼

What is a Slicer?

✓ Visual control for filtering
✓ Appears on report canvas
✓ Users click to filter

Slicer Types:

Dropdown: Compact, searchable
List: All values visible
Buttons: Single/multi-select
Date: Calendar picker
Relative Date: Last 30 days
➕ Adding Slicers▼

How to Insert:

1. Insert → Slicer
2. Select field (e.g., Region)
3. Slicer appears on canvas
4. Resize and position

Best Field Types:

✓ Categories: Region, Product
✓ Dates: Month, Year
✓ Statuses: Active, Inactive
⚙️ Slicer Formatting▼

Formatting Options:

✓ Style: Modern, Classic
✓ Orientation: Horizontal, Vertical
✓ Selection: Single, Multi
✓ Text size: Readable

Pro Tips:

Use "Select all" for quick reset
Sort values logically
Group related slicers together

Filters

🔍 Filter Pane▼

Filter Levels:

Page Filters: Affect all visuals
Visual Filters: Affect one visual
Drill Filters: Temporary, interaction

Accessing Filters:

View → Filters pane
Or: Click funnel icon
Shows all filters on page
➕ Adding Filters▼

Page Filter:

1. Open Filters pane
2. Drag field to "Page filters"
3. Set values: Include/Exclude

Visual Filter:

1. Select visual
2. Drag field to "Visual filters"
3. Configure filter type
🔧 Filter Configuration▼

Filter Types:

Basic Filtering: List of values
Advanced Filtering: AND/OR logic
Relative Date: Last N days
Top N: Top 10 products

Filter Mode:

Edit: Users can change
Lock: Fixed, users see only

Filter Types

📋 Basic Filter▼

Use Case:

✓ Single or multi-select values
✓ Example: Select regions

Configuration:

Field: Text or numeric
Operation: is, is not
Values: Checkboxes to select
🔢 Advanced Filter▼

Use Case:

✓ Complex logic: AND/OR
✓ Example: Sales > $100K AND Region = West

Operators:

is, is not, >, <, >=, <=
contains, does not contain
is blank, is not blank
📅 Date Filter▼

Types:

Relative Date: Last 30 days
Between: Date range picker
Before/After: Single date

Common Uses:

Show only current year
Last quarter's data
Year-to-date comparison
🏆 Top N Filter▼

Purpose:

Show top/bottom items only
Example: Top 10 products by sales

Configuration:

Show: Top or Bottom
Count: 5, 10, 20
By: Sales, Units, Count

Filter Interactions

🔗 Cross-Filtering▼

What is it?

✓ Clicking visual A filters visual B
✓ Interactive, dynamic exploration

How to Configure:

1. Format → Edit interactions
2. Select source visual
3. Hover over targets
4. Choose: Filter, Highlight, None

Options:

Filter: Show only matching data
Highlight: Gray out others
None: No interaction
🚫 Disabling Interactions▼

When to Disable:

✓ Prevent accidental filtering
✓ Complex dashboards
✓ Independent visuals

How:

Format → Edit interactions
Select visual → Click target
Choose "None"
🔄 Drill-Through▼

What is it?

Right-click visual → Drill-through
Navigate to detail page
Context-sensitive data passed

Setup:

1. Create detail page
2. Add drill-through filters
3. Link from main page

Real-World Scenarios

Scenario 1: Sales Dashboard

Requirements:

Filter by Region, Product, Year
Top 10 products slicer
Cross-filter: Region → Sales chart

Solution:

1. Add Region, Product dropdown slicers
2. Add Year date slicer
3. Create Top 10 filter on products
4. Enable cross-filtering
Scenario 2: KPI Report

Requirements:

Show only current quarter
Lock filters (users read-only)
No cross-filtering

Solution:

1. Add relative date filter (Q current)
2. Set filter mode: "Read-only"
3. Disable all interactions
4. Focus on key metrics
Scenario 3: Exploratory Dashboard

Requirements:

Users drill-down into details
Cross-filter all visuals
Drill-through to detail page

Solution:

1. Create main analysis page
2. Add detail page
3. Enable all interactions
4. Allow drill-through
Scenario 4: Compliance Report

Requirements:

User cannot override filters
Show only permitted regions
Audit trail required

Solution:

1. Set region filter (read-only)
2. Disable interactions
3. Lock page filters
4. Document filter logic

Best Practices

✓ Slicing & Filtering Checklist▼
□ Slicers: Clear labels, logical order
□ Filters: Document all filtering logic
□ Interactions: Intentional, not random
□ Performance: Limit data with filters
□ User Experience: Easy to understand
□ Default: Sensible defaults set
□ Drill-through: Links to relevant pages
□ Testing: Verify all filter combinations
⚠️ Common Mistakes▼
⚠ Too many slicers - Limit to 3-5
⚠ Unlabeled filters - Always label
⚠ Cross-filtering chaos - Intentional only
⚠ No default selection - Set defaults
⚠ Hidden filters - Make visible
⚠ Performance drain - Filter early
⚠ Confusing interactions - Test thoroughly
