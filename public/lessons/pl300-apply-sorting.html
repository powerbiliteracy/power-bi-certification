Apply Sorting to Visuals

Sorting controls how data appears in visuals. You can sort ascending, descending, or by custom sort order. Strategic sorting highlights important data, makes comparisons easier, and guides user attention to key insights without changing the underlying data.

Sort TypeDirectionUse Case
Ascending (A-Z)Low to highAlphabetical, chronological
Descending (Z-A)High to lowRanking, importance
CustomUser-definedNon-alphabetical order
By ValueData orderSort by related metric
💡
Key Principle
Good sorting makes patterns obvious. Lazy eyes scan top-to-bottom, so lead with the most important data.

Sorting Basics

↕️ What Is Sorting?▼

Definition:

✓ Controls display order of data
✓ Does not change actual data
✓ Applied per visual

Key Benefit:

✓ Highlights patterns visually
🎯 Why Sort Data?▼

Benefits:

✓ Makes top performers obvious
✓ Reveals outliers
✓ Improves readability
✓ Speeds insights
✓ Guides user attention
📍 Default Sort Order▼

Default Behavior:

✓ Alphabetical/chronological
✓ Based on data type
✓ Can be changed

Sort Options:

✓ Ascending (A-Z, 1-9)
✓ Descending (Z-A, 9-1)
✓ Custom order

Sort Types

🔤 Ascending Sort (A-Z)▼

What It Does:

✓ A to Z (alphabetical)
✓ 1 to 9 (numeric)
✓ Earlier to later (dates)

Good For:

✓ Alphabetical lists
✓ Chronological order
✓ Logical sequences
🔢 Descending Sort (Z-A)▼

What It Does:

✓ Z to A (reverse alphabetical)
✓ 9 to 1 (numeric high to low)
�div class="list-item">✓ Later to earlier (dates)

Good For:

✓ Rankings (sales, performance)
✓ Top performers first
✓ Most recent first
🎨 Custom Sort▼

What It Does:

✓ User-defined order
✓ Custom sort column
✓ Non-alphabetical sequence

Example:

Months: Jan, Feb, Mar (not alphabetical)
Quarters: Q1, Q2, Q3, Q4
Status: High, Medium, Low
📊 Sort by Value▼

What It Does:

✓ Sort by related metric
✓ Not by the displayed field
✓ Example: Sort products by sales

Use Cases:

✓ Rank by performance
✓ Show by revenue
✓ Order by count

Applying Sort

⚙️ Sorting from Visual▼

Quick Sort:

1. Click three dots (menu) on visual
2. Select "Sort ascending" or "Sort descending"
3. Choose field to sort by
4. Sort applied immediately

Visual Menu Options:

✓ Sort field A-Z
✓ Sort field Z-A
✓ Sort by different field
🎯 Sort via Format Panel▼

Advanced Sorting:

1. Select visual
2. Format → Sort tab
3. Multiple sort criteria
4. Set hierarchy and direction

Available Controls:

✓ Primary sort field
✓ Sort direction
✓ Secondary sort (if needed)
↔️ Multi-Level Sorting▼

Hierarchy Sorting:

1. Primary sort: Region A-Z
2. Secondary sort: Sales Z-A
3. Tertiary sort: Date A-Z

How It Works:

✓ Data grouped by first field
✓ Within each group, second sort
✓ Creates logical hierarchy

Custom Sort

🎨 Custom Sort Order▼

What Is It:

✓ Define specific order
✓ Not alphabetical or numeric
✓ Based on custom column

Example:

Month names need Jan, Feb, Mar...
Create custom sort column
Assign sort values: 1, 2, 3...
🔧 Creating Custom Sort Column▼

In Power Query:

1. Add column for sort values
2. Jan=1, Feb=2, Mar=3...
3. Back to model

In Model:

1. Select display column
2. Sort by → Select sort column
3. Visual inherits order
📊 Sort by Column▼

In Data Model:

Right-click column → Sort by column
Choose: Which column determines sort

Benefits:

✓ Persistent across all visuals
✓ Automatically applied
✓ No per-visual setup needed

Advanced Features

🔐 Sort Persistence▼

Visual-Level Sorts:

✓ Saved with report
✓ Applied by default
✓ Users can override (if enabled)

User Interactions:

✓ Readers can re-sort (if allowed)
✓ Sort reverts when refreshed
⚖️ Sorting with Aggregates▼

Aggregate Behavior:

✓ Can sort by sum, average, count
✓ Not just by individual value
✓ Example: Products by total sales

Example:

Show regions
Sort by total sales (descending)
Highest revenue regions first
🎯 Performance Considerations▼

Impact:

✓ Sorting is fast (in-memory)
✓ No performance penalty
✓ Safe to sort large datasets

Best Practice:

✓ Sort at model level when possible
✓ Reduces per-visual configuration

Real-World Scenarios

Scenario 1: Sales Rankings

Requirement:

Show top salespersons
Highest revenue first
Quick insight into performance

Solution:

Bar chart of salespeople
Sort by sales revenue descending
Top performer obvious at top
Scenario 2: Timeline Report

Requirement:

Monthly data chronologically
Jan, Feb, Mar... (not alphabetical)
Consistent across all visuals

Solution:

Create sort column: 1-12
Month column sorted by it
All visuals inherit order
Scenario 3: Status Categories

Requirement:

Show statuses: High, Medium, Low
Not alphabetically
Custom priority order

Solution:

Custom sort column
High=1, Medium=2, Low=3
Sort by this column
Scenario 4: Product Analysis

Requirement:

Products sorted by category
Within category, by revenue
Two-level sort hierarchy

Solution:

Primary sort: Category A-Z
Secondary sort: Revenue Z-A
Products grouped logically

Best Practices

✓ Sorting Design Checklist▼
□ Choose purposeful sort order
□ Descending for rankings
□ Ascending for categories
□ Custom sort for non-sequential
□ Consistent across related visuals
□ Consider mobile readability
□ Test with actual data
□ Document sort rationale
⚠️ Common Mistakes▼
⚠ Default alphabetical - Not always best
⚠ Inconsistent sorts - Use same order
⚠ No custom sort - Use when needed
⚠ Ignoring readability - Check display
⚠ Overly complex - Keep 2-3 levels max
⚠ Not testing - Test on target data
