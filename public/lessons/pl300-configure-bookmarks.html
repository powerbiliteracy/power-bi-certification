Configure Bookmarks

Bookmarks capture the current state of a report page—filters, slicers, visual selections, and focus. Users can quickly navigate between saved states, enabling guided analysis workflows and interactive storytelling.

ElementCapturesUse Case
FiltersPage/visual filtersSpecific data views
SlicersSelection stateUser selections
Visual FocusSpotlight visualHighlight key chart
PageActive pageNavigate between pages
💡
Key Principle
Bookmarks guide users through a story. Each bookmark should represent a meaningful insight or question to explore.

Bookmark Basics

🔖 What Are Bookmarks?▼

Definition:

✓ Snapshot of report page state
✓ Includes filter, slicer, selection state
✓ Reusable, shareable

What Gets Saved:

✓ Filter values (current selections)
✓ Slicer selections
✓ Visual focus (spotlight)
✓ Page context
✓ Drill status
🎯 Why Use Bookmarks?▼

Benefits:

✓ Guide users through insights
✓ Save frequently used views
✓ Reduce manual filtering
✓ Enable storytelling
✓ Improve accessibility

Use Cases:

✓ Pre-set analysis views
✓ Comparison reports
✓ Regional dashboards
✓ Tutorial walkthroughs
📍 Viewing Bookmarks Pane▼

How to Access:

View → Bookmarks → Bookmark Pane
Or: View tab → Bookmarks button

Pane Shows:

✓ All bookmarks for current page
✓ Edit/delete options
✓ Navigation buttons

Creating Bookmarks

✏️ Creating New Bookmarks▼

Steps:

1. Configure report state (filters, slicers)
2. View → Bookmarks → Add Bookmark
3. Name bookmark descriptively
4. Save

Best Practices for Naming:

✓ Clear, descriptive names
✓ Example: "Q4 Regional Sales"
✓ Avoid generic names like "Bookmark1"
🎨 Capture Options▼

What to Capture:

✓ Current Page: Active page only
✓ Display: Show/hide visuals (spotlight)

Checkboxes:

☑ Filters: Always captured
☑ Slicers: Always captured
☐ Drill: Optional drill state
🔄 Updating Bookmarks▼

Edit Existing:

1. Set report to desired state
2. Right-click bookmark → Update
3. Saves over existing bookmark

Or Create New:

1. Different state = new bookmark
2. Use "Update" to modify existing

Managing Bookmarks

🗂️ Bookmark Pane▼

Features:

✓ List all bookmarks on page
✓ Right-click context menu
✓ Double-click to navigate

Actions Available:

✓ View: Jump to bookmark
✓ Update: Modify existing
✓ Delete: Remove bookmark
✓ Rename: Change name
🔀 Ordering Bookmarks▼

Organization:

✓ Drag to reorder in pane
✓ Logical sequence for users
✓ Group by category/theme

Tips:

✓ Put "Start" or overview first
✓ Logical flow for analysis
✓ Name suggests progression
🔐 Bookmark Visibility▼

Page-Level:

✓ Each page has own bookmarks
✓ Not shared across pages

User Access:

✓ Reading mode: View bookmarks
✓ Editing mode: Create/manage
✓ Viewers can navigate saved states

Buttons & Navigation

🔘 Adding Buttons▼

Insert Button:

1. Insert → Button
2. Configure button action
3. Set to "Bookmark" action
4. Select destination bookmark

Button Options:

✓ Style: Solid, Outline, Ghost
✓ Icon: Add visual indicator
✓ Text: Clear label
⚙️ Button Actions▼

Action Types:

✓ Bookmark: Jump to saved state
✓ Drill: Navigate hierarchy
✓ Page: Go to different page
✓ Back: Return to previous state

Conditional Formatting:

✓ Show/hide button based on state
✓ Enable/disable based on filter
🎯 Navigation Flow▼

Design Patterns:

✓ Home → Detail flow
✓ Back buttons between states
✓ Next/Previous progression

Best Practice:

✓ Clear, logical progression
✓ Always provide "Home" option
✓ Label buttons descriptively

Advanced Features

🎬 Visual Display State▼

Spotlight:

✓ Focus on single visual
✓ Dim/hide others
✓ Captured in bookmark

How to Use:

1. Click visual → (icon) Spotlight
2. Save as bookmark
3. Navigating restores spotlight
🔍 Drill Status▼

Drill Capture:

✓ Save drill-down state
✓ Includes drilled level
✓ Optional in bookmark settings

Use Case:

✓ Guide users to specific drill
✓ Preserve exploration state
⚙️ Conditional Buttons▼

Visibility Rules:

✓ Show if filter = value
✓ Disable based on state
✓ Format conditionally

Example:

✓ Show "Details" only if selection
✓ Disable "Previous" on first step

Best Practices

✓ Bookmark Configuration Checklist▼
□ Clear, descriptive names
□ Logical grouping and ordering
□ Meaningful state differences
□ Consider drill and spotlight
□ Provide navigation buttons
□ Test all bookmark flows
□ Document bookmark purpose
□ Limit bookmarks per page (5-10)
⚠️ Common Mistakes▼
⚠ Too many bookmarks - Keep focused
⚠ Vague names - Be descriptive
⚠ No buttons - Add navigation
⚠ Random order - Logical flow
⚠ No home bookmark - Always include
⚠ Duplicate states - Avoid redundancy
⚠ Poor documentation - Explain purpose
