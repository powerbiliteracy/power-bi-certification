Data Loading Configuration

Data loading determines where query results go. Load to Model = visible in reports. Query Only = hidden, for transformations. This choice impacts file size, refresh speed, and usability.

AspectLoad to ModelQuery Only
VisibleYes (Fields pane)No (Hidden)
File sizeIncreasesNo impact
RelationshipsCan createCannot
MeasuresYesNo
💡
Key Principle
Load only final tables to model. Use Query Only for sources and intermediate steps. Best practice: ~80% Query Only, 20% Load to Model. This keeps models lean and files small.

Load Options

📊 Load to Model (Enabled)▼

When enabled:

✓ Imported into Power BI model as table
✓ Visible in Fields pane
✓ Can create relationships
✓ Can create measures
✓ Data stored (increases file size)

Use for:

✓ Fact tables
✓ Dimension tables
✓ Any table for reports
🔄 Query Only (Disabled Load)▼

When disabled:

✓ Results stay in Power Query
✓ NOT imported to model
✓ NOT visible in Fields pane
✓ Can reference from other queries
✓ No file size impact

Use for:

✓ Source/raw data
✓ Intermediate transformations
✓ Helper queries
⚙️ How to Change▼

In Power Query:

1. Right-click query
2. Select "Enable load" or "Disable load"
3. Checkmark = Load Enabled

In Power BI:

• Model → Query settings → Toggle
• Right-click table → Toggle load

Performance Impact

📁 File Size▼

Load to Model = adds data (compressed). Query Only = no data storage.

1M rows, 10 queries: Load All: ~200 MB Load 3 Final Tables: ~60 MB Query Only All: ~20 MB
⚡ Refresh Speed▼

More loaded tables = longer refresh. Query Only = faster (no compression).

💾 Memory Usage▼

Load to Model = more RAM. Query Only = minimal memory.

Real Scenarios

Scenario 1: Typical Model Structure
Mixed load settings for best performance

Layout:

Raw_Sales → Query Only (source) Raw_Customers → Query Only (source) CleanedSales → Query Only (intermediate) ↓ FactSales → Load to Model ✓ DimCustomer → Load to Model ✓ DimDate → Load to Model ✓

Result:

✓ Small file (~50 MB)
✓ Fast refresh
✓ Clean user experience
Scenario 2: Too Many Loaded Tables
Loads everything = performance issues

Problem:

⚠ All 15 queries loaded to model
⚠ 250 MB file (too large)
⚠ 10+ minute refresh

Solution:

✓ Keep only 3-4 final tables loaded
✓ Set rest to Query Only
✓ File: 50 MB, Refresh: 2 minutes

Best Practices

✓ Best Practices▼
✓ Load only final tables to model
✓ Set all source queries to Query Only
✓ Set intermediate steps to Query Only
✓ Keep ~80% Query Only, 20% loaded
✓ Name tables clearly (Fact_, Dim_)
✓ Delete unused queries completely
✓ Monitor file size (goal: <100MB)
