Configure the Report Page

Report pages are the canvas for your visuals. Configuring page size, layout, and properties ensures professional appearance, readability, and optimal viewing experience across devices.

SettingPurposeImpact
Page SizeDefine canvas dimensionsVisual fitting and scaling
OrientationPortrait or LandscapeWidth/height ratio
Display ModeFit to page or customScrolling behavior
BackgroundPage color/imageVisual hierarchy
Auto-fitResponsive sizingMobile compatibility
💡
Key Principle
Page configuration should balance aesthetics with functionality. Viewers should see content without excessive scrolling or cramping.

Page Size & Orientation

📏 Standard Page Sizes▼

Common Preset Sizes:

Letter: 8.5" x 11" (US standard)
A4: 210 x 297mm (ISO standard)
Tabloid: 11" x 17" (wide)
Custom: Define exact dimensions

When to Use:

✓ Letter: Standard reports, printing
✓ A4: International, PDF export
✓ Tabloid: Dashboards, wide content
✓ Custom: Special requirements
🔄 Orientation Options▼

Portrait (Vertical):

✓ Height > Width
✓ Default for reports
✓ Good for single column

Landscape (Horizontal):

✓ Width > Height
✓ Better for dashboards
✓ Multiple columns side-by-side

How to Change:

File → Page Setup (or Format Page)
Select Portrait or Landscape
⚙️ Display Mode▼

Fit to Page:

✓ All content visible without scroll
✓ Recommended for dashboards

Custom Size:

✓ Define exact width × height
✓ May require scrolling
✓ Good for detailed reports

Page Layout & Design

🎨 Page Layout Modes▼

Classic Layout:

✓ Traditional grid-based
✓ Visuals snap to grid
✓ Precise alignment

Modern Layout:

✓ Responsive, flexible positioning
✓ Better for mobile viewing
✓ Free-form placement

Switching Layouts:

View tab → Page Layout dropdown
Choose Classic or Modern
📌 Visual Placement▼

Positioning:

Drag visuals to desired location
Stack vertically for scrolling
Arrange side-by-side for width

Sizing Visuals:

Drag corners to resize
Use Format pane for exact size
Maintain aspect ratio if needed

Grouping:

Select multiple → Group together
Move as one unit
Lock position if needed
🎯 Background & Styling▼

Page Background:

Format → Page properties
Solid color or image
Transparency/opacity control

Best Practices:

✓ White or light background
✓ High contrast with text
✓ Avoid busy patterns
✓ Consistency across pages

Page Properties

⚙️ General Properties▼

Page Name:

✓ Identify page in tab
✓ Used in navigation
✓ Clear, descriptive names

Page Tab Display:

✓ Show page tab (default)
✓ Hide tab if not needed

How to Access:

Right-click page tab → Rename
Format → Page information
🔍 Visibility & Navigation▼

Visible Pages:

✓ Show in report (default)
✓ Users see in navigation

Hidden Pages:

✓ Not visible to users
✓ Still accessible via drill-through
✓ Used for detail pages

Page Order:

✓ Drag tabs to reorder
✓ Logical flow for users
📊 Sizing & Scaling▼

Page Size Options:

Aspect Ratio: Maintain proportions
Scaling: Auto-fit or custom

Responsive Behavior:

✓ Desktop: Full size
✓ Tablet: Scaled view
✓ Mobile: Stacked layout

Page Navigation

🔗 Page Navigation Pane▼

Navigation Pane:

✓ Shows all report pages
✓ Quick switching between pages
✓ Can be hidden if single page

Enable/Disable:

View → Show/Hide navigation pane
Or: View tab checkbox
📑 Page Tab Organization▼

Tab Naming:

✓ Clear, descriptive names
✓ Logical order (Overview first)
✓ Use consistent naming scheme

Tab Examples:

Overview, Sales, Inventory, Details
Dashboard, Analysis, Raw Data
Summary, Q1, Q2, Q3, Q4
🔄 Drill-Through & Bookmarks▼

Drill-Through:

Right-click visual → Drill-through
Navigate to detail page with context

Bookmarks:

✓ Save report state (filters, selections)
✓ Quick navigation to saved view
✓ View → Bookmarks pane

Mobile & Responsive Design

📱 Mobile Layout▼

Mobile Viewing Mode:

✓ Optimized for phone/tablet
✓ Vertical layout (portrait)
✓ Single column display

Features:

✓ Touch-friendly interactions
✓ Readable text/visuals
✓ Minimal horizontal scrolling
🔧 Responsive Configuration▼

Setting Up Responsive:

View → Mobile Layout (if available)
Configure mobile-specific layout
Stack visuals vertically

Best Practices:

✓ Use Fit to Page mode
✓ Vertical stacking on mobile
✓ Large touch targets (buttons)
✓ Test on actual devices
⚡ Performance on Mobile▼

Optimization:

✓ Limit visuals per page
✓ Use simpler visuals
✓ Optimize data refresh

Testing:

✓ Preview in mobile view
✓ Test on different devices
✓ Check network performance

Real-World Scenarios

Scenario 1: Executive Dashboard

Requirement:

Single page, no scrolling
Key metrics above the fold
Professional appearance

Configuration:

1. Page size: Tabloid (11 x 17)
2. Layout: Modern, responsive
3. Background: White
4. Display mode: Fit to page
Scenario 2: Multi-Page Report

Requirement:

Overview + Detail pages
Easy navigation
Drill-through capability

Configuration:

1. Overview page: Summary visuals
2. Detail pages: Hidden initially
3. Drill-through: Links overview to detail
4. Navigation: Visible page tabs
Scenario 3: Mobile Report

Requirement:

Optimized for phones
Vertical scrolling only
Touch-friendly controls

Configuration:

1. Page size: Custom (portrait)
2. Layout: Modern, stacked
3. Display: Fit to width
4. Visuals: Large, simple charts
Scenario 4: Printable Report

Requirement:

Print-friendly layout
Fits standard paper
Clear section breaks

Configuration:

1. Page size: Letter (8.5 x 11)
2. Orientation: Portrait
3. Margins: 0.5 inch
4. Colors: Print-safe (B&W capable)

Best Practices

✓ Page Configuration Checklist▼
□ Page size appropriate for content
□ Orientation matches layout needs
□ Display mode: Fit to page or custom
□ Background: Professional, consistent
□ Page names: Clear and descriptive
□ Navigation: Logical page order
□ Mobile: Responsive design tested
□ Alignment: Visuals properly spaced
⚠️ Common Mistakes▼
⚠ Cramped layout - Use larger page size
⚠ Excessive scrolling - Fit to page
⚠ Poor mobile view - Test responsiveness
⚠ Vague page names - Be specific
⚠ No navigation pane - Help users
⚠ Complex backgrounds - Keep simple
⚠ Unaligned visuals - Use grid snap
