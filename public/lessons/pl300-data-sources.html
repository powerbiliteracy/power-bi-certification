Key Concepts

📊 Data Source Types ▼

Power BI can connect to multiple data source categories:

Files:
Excel workbooks (.xlsx, .xls)
CSV, JSON, XML, PDF, Parquet, Folder
Databases:
SQL Server, MySQL, PostgreSQL, Oracle, Access
Cloud Services:
Azure SQL, Azure Synapse, Google BigQuery, Snowflake
Online Services:
Salesforce, Dynamics 365, SharePoint, Dataverse
🔗 Shared Semantic Model ▼

A semantic model (formerly 'dataset') is a single, certified data source shared across your organization.

Key Characteristics:
Stored in Power BI Service (cloud)
One source of truth for all users
Consistency across all reports and dashboards
Reduces data duplication
Enables role-based security and governance
Can be connected via 'Power BI datasets' connector
⚡ Import vs DirectQuery ▼

Choose based on your data size, refresh frequency, and performance needs:

Import Mode:
✓ Data stored in Power BI model, faster queries
✓ Larger file sizes
✓ Full DAX and transformation capabilities
DirectQuery Mode:
✓ Real-time data from source
✓ Smaller file sizes
✓ No data duplication
⚠ Slower query performance
🌉 Data Gateway ▼

Required to refresh imported data or use DirectQuery from on-premises sources.

Personal Gateway:
Single user, on their machine
For personal use only
Standard Gateway:
Multiple users, enterprise-level
Handles authentication and data encryption
Critical for hybrid environments

Hands-On Scenarios

🎯
Learn by Example
Work through real-world scenarios to master different data connection types.
Scenario 1: Excel Data to Power BI
Your manager has a customer dataset in Excel. Import it into Power BI Desktop.

Steps:

1
Open Power BI Desktop
Launch Power BI Desktop application
2
Select Home > Get Data > Excel
Choose Excel as your data source
3
Browse to the Excel file location
Navigate to your file
4
Select the worksheet containing data
Choose the specific sheet
5
Preview and Load the data
Data is now available in Power BI for modeling
Scenario 2: Connect to a Shared Semantic Model
Your organization has a certified semantic model in Power BI Service. Use it instead of importing raw data.

Steps:

1
Open Power BI Desktop
2
Select Home > Get Data > Power BI datasets
3
Authenticate with your organizational account
4
Browse and select the certified dataset
5
Create queries on top of the semantic model
You're connected to a single source of truth, ensuring data consistency
Scenario 3: Database Connection
Connect directly to your organization's SQL Server database.

Steps:

1
Select Home > Get Data > SQL Server
2
Enter server name and database name
3
Choose Direct Query or Import mode
4
Configure authentication (Windows or Database)
5
Select tables for your model
Real-time or imported data is ready for analysis
Scenario 4: Cloud Data Warehouse
Your company uses Azure Synapse. Connect to it in Power BI.

Steps:

1
Select Home > Get Data > Azure > Azure Synapse
2
Provide server and database credentials
3
Use Service Principal or user authentication
4
Select tables from the data warehouse
5
Decide between Import and Direct Query modes
You have access to cloud-scale analytics infrastructure
