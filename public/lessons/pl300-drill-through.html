Configure Drill Through Navigation

Drill through enables right-click navigation from summary data to detail pages. The selected data context automatically filters the detail page, enabling guided deep-dive analysis without manual selection.

Setting Up Drill Through

🛠️ Configuring Detail Page▼

Steps:

1. Create or navigate to detail page
2. Right-click page tab
3. Select "Page type" or properties
4. Enable drill through option
5. Specify fields to receive context

Alternative:

Format → Page → Drill through → Enable
📋 Specifying Fields▼

Which Fields to Pass:

1. Click "Add drill through field"
2. Select field from data model
3. This field receives context value
4. Page auto-filters on this field

Example:

Source page: Click on "Product A"
Pass: Product field
Detail page filters: Product = A
🔗 Mapping Fields▼

Field Matching:

✓ Auto-map common field names
✓ Manual mapping if different
✓ Use consistent naming

Important:

✓ Source and target must match
✓ Data types should be same
✓ Test field mapping

Context Passing

🎯 How Context Works▼

Flow:

1. User right-clicks data point
2. Drill through menu appears
3. User selects target page
4. Page opens with filters applied

Auto-Filtering:

✓ No user action needed
✓ All configured fields filter
✓ Visuals show filtered data
📊 Multiple Fields▼

Passing Multiple Values:

✓ Configure multiple fields
✓ All values pass together
✓ Example: Region + Product

Detail Page Behavior:

✓ Filters on all fields
✓ AND logic (both conditions)
✓ Shows intersection
⚙️ Cross-Table Passing▼

Relationships:

✓ Pass through relationships
✓ Auto-filter related tables
✓ Example: Customer → Orders

Requirements:

✓ Tables must be related
✓ Relationship defined
✓ Field matching needed
