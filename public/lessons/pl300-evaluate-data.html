The latest PL-300 syllabus calls this out explicitly under Profile and clean the data → Evaluate data, including data statistics and column properties. You need to know how to surface profiling info in the Power Query editor and what to do with it.

Three profiling tools in Power Query

📐 Column quality▼
View → Column quality (or thin coloured bar above each column header).
Shows Valid / Error / Empty percentages.
First red flag: high Error % means a type mismatch or bad source data; high Empty % means nulls.
📈 Column distribution▼
View → Column distribution.
Shows Distinct count and Unique count per column.
Use it to confirm a key column is truly unique (Distinct = Unique = row count).
📊 Column profile▼
View → Column profile (best with a single column selected).
Numeric columns: count, min, max, average, standard deviation, even/odd, zero, distinct.
Text columns: shortest/longest length, character distribution.
Value distribution chart shows the most frequent values.
🔢 Profile based on top 1,000 rows vs entire data set▼
Bottom-left status bar lets you switch between Top 1000 rows (default, fast) and Entire data set.
Always switch to the entire data set before publishing — top 1,000 can lie about distinct counts and quality.

⚠️
Set types in Power Query, not the model
Always change data types upstream in Power Query, not on the loaded model. Type changes in the model can silently break refresh.

Scenarios

Scenario 1 — Customer ID column should be a key but Distinct ≠ Unique
Diagnosis: The column has duplicates. Switch to "Profile based on entire data set" to confirm, then de-duplicate or pick a different key.
Scenario 2 — Sales column shows 7% Errors
Diagnosis: Column quality reveals errors. Use "Keep Errors" to inspect them — usually text values where numbers were expected. Fix in source or replace value before changing type.
Scenario 3 — Address column doesn't show on a map visual
Fix: In Model view set Data category = Address (or City/Country). Power BI uses this to geocode for map visuals.
Scenario 4 — Months sort alphabetically (April, August, December…)
Fix: In Model view, set Month Name's "Sort by column" property to MonthNumber. The visual will then sort correctly.

Best practices

✅ Always profile before modelling▼
Turn on all three profiling views (View tab) on day one.
Switch to "Entire data set" before signing off on the model.
✅ Lock down data types & locale▼
Change types explicitly in Power Query — never rely on type detection alone.
Use "Change Type → Using Locale" for international date/number formats.
✅ Set model-level properties▼
Hide keys, set Summarize by = Don't summarize on IDs, set Data category for geo / URL / image columns.
