Dimensional Modeling Fundamentals

Fact tables store measurable events (sales, transactions). Dimension tables store context. Together they form a star schema for analytics.

AspectFact TableDimension Table
PurposeMeasures & eventsAttributes & context
RowsMany (millions)Few (thousands)
ColumnsFew measuresMany attributes
ExampleFactSalesDimProduct, DimDate
💡
Key Principle
Star schema optimizes analytics by separating measures from attributes. Fact tables connect to multiple dimensions via foreign keys, enabling fast queries and dimensional slicing.

Star Schema Design

⭐ Fact Table Structure ▼

Central table with measurable events and foreign keys.

FactSales ├── SalesID (PK) ├── DateID (FK) ├── ProductID (FK) ├── CustomerID (FK) ├── Amount (Measure) ├── Quantity (Measure) └── Profit (Measure)

Characteristics: Many rows, multiple FKs, additive measures

🔵 Dimension Table Structure ▼

Context and descriptive attributes.

DimProduct ├── ProductID (PK) ├── ProductName ├── Category ├── Subcategory ├── Price └── Supplier DimDate ├── DateID (PK) ├── FullDate ├── Year, Month, Quarter ├── DayOfWeek └── IsWeekday

Characteristics: Fewer rows, many attributes, no FKs (except PK)

🔗 Grain & Relationships ▼

Grain: Level of detail for each fact row (e.g., one row per transaction, per day, per region).

Relationships: Many-to-One from fact to each dimension via foreign keys.

FactSales (Grain: per transaction) ├─→ DimDate (Many transactions per date) ├─→ DimProduct (Many transactions per product) ├─→ DimCustomer (Many transactions per customer) └─→ DimRegion (Many transactions per region)

Real-World Scenarios

Scenario 1: Retail Sales Data Warehouse
Store transactions across multiple locations and products

Fact Table: FactSales

SalesID, DateID, ProductID, StoreID, CustomerID, Amount, Quantity, DiscountAmount

Dimension Tables:

• DimDate: Date, Month, Year, Quarter, DayOfWeek
• DimProduct: ProductName, Category, Price
• DimStore: StoreName, City, Region, Manager
• DimCustomer: CustomerName, Segment, Loyalty
Scenario 2: Website Analytics
Track page views and user interactions

Fact Table: FactPageViews

PageViewID, DateID, TimeID, UserID, PageID, Views, Bounces

Dimensions:

• DimDate (Year, Month, Week)
• DimTime (Hour, Minute)
• DimUser (UserID, Region, Device)
• DimPage (PageName, Category, Type)
