Configure Item-Level Access

Item-level access enables granular control beyond workspace roles. Configure specific permissions for individual reports, dashboards, and datasets—allowing you to share selectively, protect sensitive content, and manage access at the item level rather than workspace level.

Control LevelScopeGranularity
Workspace RoleEntire workspaceBroad
Item SharingIndividual itemsFine-grained
RLSDataset rowsFinest
💡
Key Principle
Item-level access adds granularity on top of workspace roles. Share specific items with individuals or groups while keeping workspace broad.

Item-Level Access Basics

🔒 What Is Item-Level Access?▼

Definition:

✓ Permissions at individual item level
✓ Beyond workspace role
✓ Specific users/groups

Purpose:

✓ Share selectively
✓ Protect sensitive items
✓ Fine-grained control
🎯 Item vs Workspace Access▼

Workspace Role:

✓ Access all items in workspace
✓ Single permission level

Item Access:

✓ Access specific items
✓ Different per item
✓ Overrides workspace role
🔐 Who Can Configure Item Access?▼

Permission Requirements:

✓ Item owner (typically)
✓ Workspace admin
✓ Member with edit access

Note:

✓ Workspace viewers cannot share
✓ Contributors can share reports

Item Types and Access

📊 Reports▼

Access Options:

✓ Share with users/groups
✓ View, Edit, Re-share permissions
✓ Can grant edit access

Configuration:

✓ Report → Share
✓ Enter recipient email
✓ Select permission level
📈 Dashboards▼

Access Options:

✓ Share dashboard directly
✓ View-only by default
✓ Can grant edit access

Note:

✓ Underlying report access separate
✓ Need both for full access
💾 Datasets▼

Access Options:

✓ Build permission
✓ Read permission
✓ Granular dataset sharing

Use Case:

✓ Allow others to build on data
✓ Restrict to specific teams
📄 Paginated Reports▼

Access Options:

✓ Share like regular reports
✓ View-only typical

Configuration:

✓ Same sharing interface
✓ Recipient-based access

Sharing and Permissions

👥 How to Share Items▼

Steps:

1. Open report/dashboard
2. Click Share button
3. Enter email or name
4. Select permission level
5. Send/Add

Recipients:

✓ Individual users
✓ Security groups
✓ Distribution lists
🔑 Permission Levels▼

View:

✓ Read-only access
✓ Cannot edit

Edit:

✓ Modify report/dashboard
✓ Save changes

Re-share:

✓ Share with others
✓ Only if granted
📧 Direct Email Links▼

How It Works:

✓ Email with link sent
✓ Recipient clicks link
✓ Access granted automatically

Message:

✓ Optional note to recipient
✓ Explain context

Configuration

⚙️ Configuring Item Access▼

For Reports:

1. Report → Share
2. Grant access form
3. Choose recipients
4. Select View/Edit/Re-share

For Dashboards:

1. Dashboard → Share
2. Same process
3. Note: Report access separate
📋 Viewing Current Access▼

Check Who Has Access:

✓ Item → Share/Manage access
✓ See current recipients
✓ View their permissions

Modify:

✓ Change permission level
✓ Remove access
✓ Add more users
🔄 Revoking Access▼

Process:

1. Item access management
2. Locate user/group
3. Click remove
4. Confirm removal

Effect:

✓ Immediate loss of access
✓ Cannot access item

Advanced Access Control

🔐 Build Permission (Datasets)▼

What It Does:

✓ Allow others to create reports
✓ On your dataset
✓ Self-service analytics

Configuration:

✓ Dataset → Grant build access
✓ Users can now create reports
🔗 Link Sharing vs User Sharing▼

User Sharing:

✓ Specific people/groups
✓ Email addresses required
✓ Better tracking

Link Sharing:

✓ Anyone with link
✓ Easier distribution
✓ Less control
⚡ Cascading Permissions▼

Important:

✓ Workspace role + item access
✓ Most restrictive applies
✓ Both must grant access

Example:

✓ Workspace Viewer + item shared
✓ Can view item (viewer elevated)

Real-World Scenarios

Scenario 1: Sensitive Financial Report

Requirement:

Confidential financial data
Only CFO and controller
In shared team workspace

Solution:

Share report to CFO and controller
View-only permission
Others in workspace cannot see
Scenario 2: Self-Service Dataset

Requirement:

Sales data available
Analysts create own reports
Controlled dataset

Solution:

Grant build permission
To sales analyst group
They create reports independently
Scenario 3: Collaborative Report

Requirement:

Team edits same report
Multiple editors
Controlled access

Solution:

Share report with Edit permission
To team members
All can modify
Scenario 4: Executive Dashboard

Requirement:

Dashboard for executives
Workspace has many users
Only execs see dashboard

Solution:

Share dashboard to exec group
View-only
Others don't have access

Best Practices

✓ Access Management Strategy▼
□ Start restrictive, expand as needed
□ Document access decisions
□ Use groups for teams
□ Review access regularly
□ Remove unused access
□ Limit edit access
□ Communicate changes
□ Audit for compliance
⚠️ Common Mistakes▼
⚠ Over-sharing - Share minimally
⚠ Edit to everyone - Restrict editing
⚠ Forgetting report access - Check both
⚠ Not removing old access - Cleanup
⚠ Individual vs group - Use groups
⚠ No documentation - Document why
⚠ Ignoring RLS - Combine with RLS
