Manage and Publish Reports

Publishing reports moves them from Desktop to Power BI Service for cloud sharing. You can publish to workspaces, manage permissions, refresh schedules, configure alerts, and monitor report usage—enabling team collaboration and governance.

EnvironmentPurposeAccess
DesktopDevelopment/editingLocal only
ServiceSharing and collaborationCloud/web
WorkspaceTeam organizationGroup-based
AppDistributionPublic sharing
💡
Key Principle
Publish early, manage continuously. Reports live in Service, not Desktop.

Publishing Basics

📤 What Is Publishing?▼

Definition:

✓ Upload report to Power BI Service
✓ Make available in cloud
✓ Enable sharing and collaboration

Result:

✓ Web access
✓ Mobile access
✓ Refresh schedules
🏠 Desktop vs Service▼

Desktop:

✓ Development environment
✓ Edit reports/data
✓ Local file (.pbix)

Service:

✓ Production environment
✓ Shared viewing/collaboration
✓ Cloud-hosted
🎯 Why Publish?▼

Benefits:

✓ Share with teams
✓ Access from anywhere
✓ Auto-refresh data
✓ Set permissions
✓ Monitor usage

Publishing Process

🚀 Publishing Steps▼

How to Publish:

1. Home → Publish
2. Or: File → Publish
3. Select destination workspace
4. Click Publish
5. Confirm upload

Result:

✓ Report in Service
✓ Dataset uploaded
✓ Ready for sharing
📊 What Gets Published?▼

Published Items:

✓ Report (.pbix)
✓ Dataset
✓ Related queries

Not Published:

✓ Local queries
✓ Power BI Service-only items
🔄 Publishing Updates▼

Update Published Report:

1. Make changes in Desktop
2. Click Publish
3. Choose: Replace or New
4. Confirm replacement

Result:

✓ Service report updated
✓ Users see new version

Workspaces

📁 What Are Workspaces?▼

Definition:

✓ Container for related content
✓ Team collaboration space
✓ Organize by project/team

Types:

✓ My Workspace (personal)
✓ Team Workspaces (shared)
✓ Classic vs Modern
👥 Workspace Roles▼

Member Roles:

✓ Admin: Full control
✓ Member: Create/edit
✓ Contributor: Create/edit
✓ Viewer: View only

Permissions:

✓ Each role has capabilities
✓ Set per user
⚙️ Workspace Management▼

Setup:

1. Service → Workspaces
2. Create New
3. Name and description
4. Add members

Settings:

✓ Name and description
✓ Image/branding
✓ Member access

Sharing & Access Control

🔗 Sharing Methods▼

Share Report:

1. Service → Report
2. Share button (top)
3. Enter email addresses
4. Set permissions

Share Dashboard:

✓ Same process
✓ Can share filtered views
🔐 Permission Levels▼

Share Permissions:

✓ View: Read-only access
✓ Edit: Modify content
✓ Share: Can reshare

Dataset Permissions:

✓ Read: Access data
✓ Build: Create new reports
✓ Share: Reshare dataset
📱 Apps▼

What Are Apps:

✓ Curated content bundles
✓ From workspace
✓ For distribution

Create App:

1. Workspace settings
2. Create app version
3. Publish to users

Management

🔄 Refresh Schedules▼

Configure Refresh:

1. Service → Dataset
2. Settings → Refresh
3. Set schedule
4. Frequency: Daily/Weekly

Options:

✓ Basic: 8 refreshes/day
✓ Premium: 48 refreshes/day
⚡ Alerts & Notifications▼

Create Alerts:

1. Card/gauge visual
2. Click → Set alert
3. Condition (>500)
4. Email notification

Use Case:

✓ Monitor KPIs
✓ Alert on thresholds
📊 Usage Analytics▼

Monitor Usage:

✓ Service → Reports
✓ Usage analytics
✓ See who accesses

Metrics:

✓ Views and shares
✓ User engagement
✓ Performance data

Real-World Scenarios

Scenario 1: Team Collaboration

Requirement:

Team shares reports
Edit and update together
Centralized workspace

Solution:

Create team workspace
Publish reports there
Set members to Edit/Contributor
Scenario 2: Executive Dashboard

Requirement:

Share with leadership
View-only access
No editing

Solution:

Share report directly
Set permission to View
Or create read-only app
Scenario 3: Organization-Wide Release

Requirement:

Deploy to entire company
Consistent access
Managed distribution

Solution:

Create workspace
Build and test content
Publish as app
Share app to organization
Scenario 4: Data Governance

Requirement:

Control who builds
Manage dataset access
Monitor usage

Solution:

Workspace with admins
Dataset share with Build
Usage analytics tracking

Best Practices

✓ Publishing Checklist▼
□ Test thoroughly before publish
□ Plan workspace structure
□ Document workspace purpose
□ Set refresh schedules
□ Configure alerts
□ Set appropriate permissions
□ Monitor usage
□ Regular updates
⚠️ Common Mistakes▼
⚠ No refresh schedule - Set daily
⚠ Over-sharing - Use View role
⚠ Poor naming - Clear workspace names
⚠ No documentation - Document purpose
⚠ Ignoring usage - Monitor activity
⚠ Stale content - Update regularly
⚠ No alerts - Set thresholds
