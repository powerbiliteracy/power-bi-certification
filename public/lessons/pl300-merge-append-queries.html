Merge and Append Fundamentals

Merge combines queries horizontally (adds columns). Append combines vertically (adds rows).

AspectMergeAppend
DirectionHorizontal (columns)Vertical (rows)
ResultMore columnsMore rows
Join KeyMatching columnSame schema
Join TypesInner, Left, Right, FullN/A
ExampleAdd names to salesStack Q1+Q2+Q3
💡
Key Principle
Merge for enriching (more attributes). Append for combining (more records). Choose based on data structure needs.

Merge vs Append Operations

🔗 Merge: Add Columns from Another Table ▼

Combines tables horizontally using matching key.

Steps:

1
Select Merge from Home tab
Choose which table to merge with
2
Select join key columns
Match columns that link tables (e.g., CustomerID)
3
Choose join type
Inner, Left, Right, or Full Outer
4
Select columns to keep
Choose which columns to include from merged table

When to Use:

✓ Add customer info to sales transactions
✓ Combine lookup tables with fact data
✓ Enrich with attributes
📚 Append: Stack Rows from Multiple Tables ▼

Combines tables vertically by stacking rows.

Steps:

1
Ensure tables have same schema
Same column names and types
2
Select Append Queries
Choose "Append Queries" from Home tab
3
Select queries to append
Can add 2+ queries at once
4
Review combined result
All rows from all tables combined

When to Use:

✓ Combine quarterly/monthly data
✓ Stack data from multiple files
✓ Add new records to existing data

Merge Join Types

➡️ Inner Join (Matches Only) ▼

Returns only matching rows from both tables.

Use: Only rows with matching keys in both tables

❌ Drops unmatched rows from both tables

⬅️ Left Outer Join (Keep Left) ▼

Returns all rows from left table + matching from right.

Use: All your primary data + enriched where possible

Unmatched rows show NULL in merged columns

➡️ Right Outer Join (Keep Right) ▼

Returns all rows from right table + matching from left.

Use: Less common; prefer Left Outer

Mirror of Left Outer (opposite)

⬌ Full Outer Join (Keep All) ▼

Returns all rows from both tables.

Use: Keep everything, fill gaps with NULL

Best for reconciliation and completeness

Best Practices

✓ Merge Best Practices ▼
✓ Verify join key columns match in both tables
✓ Check data types are compatible
✓ Use Left Outer by default (preserves primary table)
✓ Remove duplicate columns after merge
✓ Validate result row count matches expectations
✓ Be aware of many-to-many (causes row explosion)
✓ Append Best Practices ▼
✓ Ensure identical column names in all tables
✓ Verify same data types for matching columns
✓ Use dynamic source for scalability (folders)
✓ Add source column to track origin
✓ Order queries logically (chronological, alphabetical)
✓ Test with small dataset first
⚠️ Common Mistakes ▼

Merge Mistakes:

⚠ Using Inner Join when Left Outer needed (lose data)
⚠ Merging on mismatched data types
⚠ Many-to-many relationships (exponential rows)

Append Mistakes:

⚠ Different column names between tables
⚠ Appending incompatible schemas
⚠ No way to track data source
