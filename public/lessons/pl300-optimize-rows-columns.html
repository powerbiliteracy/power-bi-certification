Optimize Performance by Removing Rows & Columns

Performance optimization removes unnecessary data: duplicate rows, unused columns, test data, archived periods. Reduces file size (MB), improves refresh time (minutes), enables query folding.

Key Principle
Optimization = Less data = Faster refresh. Remove rows (filters, duplicates) and columns (unused) early in Power Query. Enable query folding to push processing to source. Results: 50-80% smaller files, 5-10x faster refresh.

Remove Unnecessary Rows

🗑️ Identify Unnecessary Rows▼

Types to Remove:

✓ Test/dummy data (QA rows)
✓ Archived data (before cutoff date)
✓ Draft/incomplete records
✓ Null/blank key columns
✓ Duplicate rows (same ID, date)
✓ Status = "Deleted" or "Inactive"
📅 Filter by Date Range▼

Remove Old Data:

• Only load last 5 years (not 20 years)
• Archive historical in separate model

Impact:

✓ 20 year data → 5 year = 75% smaller
✓ 2 hour refresh → 30 min
🚫 Remove Test Records▼

Common Test Data:

• Customer ID = 99999 (test customer)
• Amount = 0 or negative
• Product = "TEST-*"
• Status = "Draft" or "Cancelled"
🔄 Remove Duplicates▼

Scenario:

• Same transaction ID appears 3x
• Need distinct records only

Impact:

✓ 1M rows → 800K unique (20% smaller)

Remove Unnecessary Columns

📊 Identify Unnecessary Columns▼

Types to Remove:

✓ Not used in reports/measures
✓ Duplicate columns (City + CityName)
✓ Technical columns (ETL_LoadDate)
✓ Sensitive data (SSN, credit cards)
🔍 Audit Current Columns▼

Decision Framework:

✓ Ask: "Will users filter/view this?"
✓ Ask: "Used in any measure?"
✓ Ask: "Used in relationships?"
✓ If NO to all → Remove it
💾 File Size Impact▼

Example: FactSales (1M rows)

• Original: 30 columns, 500 MB
• Remove 15 unused: 250 MB
• 50% size reduction!

Refresh Time:

• 500 MB: 30 min refresh
• 250 MB: 15 min refresh

Query Folding for Performance

⚡ What is Query Folding?▼

Definition:

✓ Power Query pushes operations to source
✓ Database does filtering/sorting
✓ Less data transferred to Power BI

Example:

Without: Pull 1M rows → Filter in Power Query
With: Query sends filter → Source returns 50K rows
✅ Query Folding Best Practices▼

Operations that Fold:

✓ Filter rows (WHERE clause)
✓ Select columns (SELECT clause)
✓ Sort rows (ORDER BY)
✓ Group & aggregate

Operations that DON'T Fold:

⚠ Custom columns (calculated cols)
⚠ Merge queries
⚠ Append queries

Strategy:

✓ Filter & select early (before custom cols)
✓ Add calculated columns last

Best Practices

✓ Optimization Best Practices▼
✓ Remove rows early in Power Query
✓ Remove columns before transformations
✓ Keep only columns needed for reports
✓ Filter dates to relevant range
✓ Remove duplicates by key columns
✓ Enable query folding (filter before custom cols)
✓ Archive old data separately
✓ Document what was removed
⚠️ Common Mistakes▼
⚠ Not removing test/duplicate rows
⚠ Deleting columns used in relationships
⚠ Custom columns before filtering (no folding)
⚠ Loading all historical data unnecessarily
