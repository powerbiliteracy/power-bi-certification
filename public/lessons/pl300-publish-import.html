Publish, Import, or Update Items in a Workspace

Content management in workspaces involves publishing from Desktop, importing files from sources, and updating existing items. You can move reports, refresh datasets, manage versions, and maintain content lifecycle—enabling smooth content deployment and updates.

ActionSourceDestination
PublishPower BI DesktopWorkspace
ImportFile (.pbix)Workspace

RefreshDatasetLatest data
💡
Key Principle
Move content efficiently. Keep items current and organized.

Basics

📤 What is Publishing?▼

Definition:

✓ Send from Desktop to Service
✓ Create workspace items
✓ Make available to others

Includes:

✓ Reports
✓ Dashboards
✓ Datasets
📥 What is Importing?▼

Definition:

✓ Load files into workspace
✓ From .pbix files
✓ Or from templates

Sources:

✓ Local files
✓ Shared drives
✓ OneDrive/SharePoint
🔄 What is Updating?▼

Definition:

✓ Modify existing content
✓ Replace or refresh
✓ New versions

Types:

✓ Data refresh
✓ Full replacement
✓ Incremental updates

Publishing

📤 Publishing Process▼

Steps in Desktop:

1. File → Publish
2. Select workspace
3. Choose destination
4. Click Publish

Result:

✓ Report in Service
✓ Dataset created
✓ Available to workspace
🎯 Choosing Workspace▼

Options:

✓ My Workspace (personal)
✓ Team workspace
✓ Department workspace

Consideration:

✓ Need publish access
✓ Member or Admin role
📊 First vs Subsequent Publish▼

First Publish:

✓ Creates all items
✓ New report/dataset

Later Publishes:

✓ Updates existing report
✓ Can replace dataset

Importing

📥 Importing Files▼

Method 1 - Upload:

1. Workspace → Upload
2. Select .pbix file
3. Click Open

Method 2 - Get Data:

1. Service → Get data
✓ Choose file source
🗂️ File Sources▼

Supported:

✓ Local .pbix files
✓ OneDrive/SharePoint
✓ Local uploads

Note:

✓ Large files may be slow
✓ Plan accordingly
⚙️ Import Settings▼

During Import:

✓ File name becomes report
✓ Dataset created
✓ Automatic refresh schedule

After Import:

✓ Configure refresh schedule
✓ Set credentials

Updating Items

🔄 Update from Desktop▼

Process:

1. Make changes in Desktop
2. File → Publish
3. Select same workspace
4. Replace existing

Result:

✓ Report updated
✓ Dataset refreshed
✓ New version
🔄 Data Refresh▼

What It Does:

✓ Refreshes underlying data
✓ Queries run again
�div class="list-item">✓ Latest values loaded

Configure:

✓ Workspace → Dataset
✓ Settings → Refresh
📊 Replace Dataset▼

Options:

✓ Update from Desktop
✓ Upload new file
✓ Complete replacement

Caution:

✓ Breaks existing reports
✓ Need same structure

Advanced

⚙️ Gateway Configuration▼

For On-Premises Data:

✓ Install gateway
✓ Configure credentials
✓ Set up refresh

Purpose:

✓ Access local databases
✓ On-premises servers
📊 Incremental Refresh▼

What It Is:

✓ Update only new data
✓ Not full refresh
✓ Faster, efficient

Benefits:

✓ Reduced load
✓ Faster updates
🔐 Version Control▼

Consider:

✓ Save as different name
✓ Keep archive copies
✓ Document versions

Best Practice:

✓ Backup before update
✓ Test in staging first

Real-World Scenarios

Scenario 1: Initial Report Launch

Requirement:

First time publishing report
From Desktop to Service
Share with team

Solution:

Open Desktop file
File → Publish
Choose team workspace
Report available to all
Scenario 2: Regular Updates

Requirement:

Weekly report changes
Keep Service current
Users see latest

Solution:

Edit in Desktop
Publish (replaces)
Users auto-refreshed
Scenario 3: Data Refresh

Requirement:

Daily data updates
No design changes
Automatic schedule

Solution:

Configure refresh schedule
Set dataset refresh daily
Reports auto-update
Scenario 4: Migrate Files

Requirement:

Move reports from old to new
From archive files
Organize in workspace

Solution:

Workspace → Upload
Import .pbix files
Configure settings

Best Practices

✓ Publishing Checklist▼
□ Test in Desktop first
□ Verify data connections
□ Check report visuals
□ Name clearly
□ Choose correct workspace
□ Document data source
□ Test in Service
□ Announce to users
⚠️ Common Mistakes▼
⚠ Wrong workspace - Verify destination
⚠ Broken connections - Test first
⚠ No refresh schedule - Configure it
⚠ Credentials issues - Set up properly
⚠ Overwriting old - Back up first
⚠ No documentation - Document sources
⚠ Large files - Optimize before publish
