Configure Navigation for a Report

Report navigation guides users through your content. You can control the page navigation pane, use buttons for drill-through, hide pages, and create intuitive report structures that prevent users from getting lost and enhance discoverability.

Key Principle
Good navigation is invisible. Users should intuitively know where to go and what to do next.

Key Components:

✓ Navigation pane (automatic)
✓ Buttons (custom)
✓ Page visibility settings
✓ Bookmarks

Benefits:

Navigation Types

📑 Navigation Pane▼

What It Is:

✓ Vertical pane with all pages
✓ Click page name to navigate
✓ Shows page hierarchy

Control Options:

✓ Show/hide pane
✓ Hide specific pages
✓ Reorder pages
🔘 Button Navigation▼

What It Is:

✓ Custom buttons on page
✓ Set button action to page
✓ Full visual control

Advantages:

✓ Branded, professional look
✓ Guided user flow
✓ Can hide pane entirely
🚫 Hidden Pages▼

What It Is:

✓ Pages not visible in pane
✓ Accessed via drill-through
✓ Can be tooltip pages

Use Cases:

✓ Detail/support pages
✓ Tooltip pages
✓ Temporary pages

Page Navigation

📄 Managing Page Visibility▼

Hide a Page:

1. Right-click page tab
2. Select "Hide page"
3. Page removed from pane
4. Still accessible via drill-through

Show Hidden Page:

1. Right-click hidden page
2. Select "Show page"
3. Returns to pane
🔄 Organizing Pages▼

Reorder Pages:

1. Right-click page tab
2. Select "Move right/left"
3. Or drag to new position

Naming Convention:

✓ Clear, descriptive names
✓ Indicate hierarchy: "Sales > By Region"
✓ Prefix with number for order
⚙️ Pane Settings▼

Control Pane Display:

File → Options → Report settings
Or: View → Navigation pane

Options:

✓ Show navigation pane: On/Off
✓ Default page on open
✓ Display options

Button Navigation

🔘 Creating Navigation Buttons▼

Insert Button:

1. Insert → Button
2. Drag to place on page
3. Format button appearance
4. Set action → Page navigation

Button Settings:

✓ Style: Solid, Outline, Ghost
✓ Icon: Choose visual indicator
✓ Text: Button label
⚙️ Button Actions▼

Action Types:

✓ Page: Navigate to page
✓ Bookmark: Go to bookmark
✓ Web URL: Open external link
✓ Back: Return to previous page

Configuration:

1. Select button
2. Format → Button → Action
3. Choose: Page action
4. Select destination page

Advanced Features

📑 Drill-Through Navigation▼

What Is Drill-Through:

✓ Right-click → Detail page
✓ Context-aware navigation
✓ Passes filter context

Setup:

1. Target page → Format
2. Drill-through → Enable
3. Specify fields to pass
🔐 Page Visibility Rules▼

Conditional Display:

✓ Show page if condition met
✓ Based on filter value
✓ Create dynamic navigation

Example:

✓ Show "Admin" page if user=admin
✓ Show "Details" only if selected
🎬 Breadcrumb Navigation▼

Implementation:

✓ Text boxes showing path
✓ Buttons for parent pages
✓ Manual creation (no built-in)

Benefits:

✓ Shows user location
✓ Easy backtracking
✓ Professional UX

Best Practices

✓ Navigation Design Checklist▼
□ Clear, logical page structure
□ Consistent navigation method
□ Easy return to home/overview
□ Descriptive page names
□ Hide non-essential pages
□ Test all navigation paths
□ Mobile-friendly design
□ Document navigation flow
⚠️ Common Mistakes▼
⚠ Too many pages - Limit to 5-7
⚠ Unclear names - Be descriptive
⚠ No back button - Always provide
⚠ Inconsistent style - Match theme
⚠ Confusing flow - Test with users
⚠ Hidden pages unclear - Document
⚠ Small buttons - Make clickable
