Handling nulls and blanks

🔍 Detect nulls fast▼
View → Column quality. The Empty % chip flags null/blank columns.
Click the column header → Filter → Remove Empty to drop them, or click the null chip to inspect.
🔄 Replace Values▼
Right-click column → Replace Values. Tick "Special values → null" to replace null with 0, "Unknown", etc.
For text columns, replacing null with "Unknown" prevents blanks in slicers and visuals.
⬇️ Fill Down / Fill Up▼
Transform → Fill → Down/Up.
Useful when an Excel-style report leaves group headers null on subsequent rows.
❌ Remove Empty rows▼
Home → Remove Rows → Remove Blank Rows (drops rows where every column is null/empty string).

Standardising inconsistent values

ProblemPower Query fix
Leading/trailing spacesTransform → Format → Trim
Inconsistent case ("usa", "USA", "Usa")Format → UPPERCASE / lowercase / Capitalize Each Word
Hidden non-printable charactersFormat → Clean (removes 0x00–0x1F)
Multiple spellings ("USA", "U.S.A", "United States")Replace Values (or merge with a mapping table)
Mixed delimiters in one columnSplit Column → By Delimiter
Numbers stored as textChange Data Type to whole/decimal number
Inconsistent date formats across regionsChange Type → Using Locale
🧩
When to use a mapping table
If you have many variants of the same value (countries, product codes, suppliers), don't chain Replace Values. Build a small mapping table and merge it in — it's transparent, testable, and easier to update.

Spotting and handling unexpected values

🔢 Numeric outliers▼
Use Column profile → min, max, mean, std-dev to spot impossible values (negative quantities, ages of 999).
Decide: replace, filter out, or flag with a calculated column for review.
📅 Date outliers▼
Look for placeholder dates like 1900-01-01 or 9999-12-31 in the value distribution chart.
Replace with null (and handle null in DAX) or filter out.
♻️ Duplicates▼
Right-click key column → Remove Duplicates (or Keep Duplicates to inspect them first).
For composite keys, select multiple columns before Remove Duplicates.
⚠️ Errors as data quality signal▼
Right-click column → Replace Errors (with null or a default) — keeps refresh succeeding.
Or Home → Keep Rows → Keep Errors to triage them in a separate quality query.

Scenarios

Scenario 1 — Country column has "USA", "U.S.A.", "United States" and " usa "
Steps: Trim → lowercase → Replace Values to a single canonical "United States" (or join to a country mapping table).
Scenario 2 — Sales table has 12% null Region after merging with Customers
Cause: Customer keys missing in the Customers table. Either clean the source, or replace null with "Unknown" so the row still shows up in totals.
Scenario 3 — Age column ranges from 0 to 999
Steps: Use Column profile to confirm. Filter Age > 120 to inspect, then either replace with null or exclude. Document the rule.
Scenario 4 — Excel export has empty cells where group headers were merged
Steps: Use Transform → Fill → Down on the group column to propagate the header value down each group.

Best practices

✅ Clean upstream when you can▼
Pushing fixes back to the source removes duplication and benefits everyone.
Use Power Query as the safety net, not the only line of defence.
✅ Prefer Replace Values over chained IFs▼
Use Replace Values, mapping tables, or conditional columns — they're easier to audit than nested if/else logic.
✅ Document the rule▼
Rename steps to describe what they do ("Replace null Region with Unknown").
Add a description in step properties so future developers know why the rule exists.
✅ Don't silently drop rows▼
If you filter out bad rows, build a parallel "quality" query that captures them so issues are visible, not hidden.
