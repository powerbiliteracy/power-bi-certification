Configure Row-Level Security Group Membership

RLS group membership assigns users to roles for automatic data filtering. Configure user-to-role mapping through Azure AD groups or manual assignment—enabling scalable, maintainable RLS where users automatically see data relevant to their assigned role without report duplication.

User Mapping Strategies

🗺️ Mapping Table Approach▼

How It Works:

✓ Create mapping table
✓ Columns: User, Region, Territory
✓ RLS rule looks up user

Benefits:

✓ Easy maintenance
✓ No model changes
✓ Flexible assignments
📊 Mapping Table Structure▼

Example Table:

✓ UserEmail | Manager | Territory
✓ john@company.com | Yes | West
✓ mary@company.com | No | East

Usage in DAX:

✓ LOOKUPVALUE looks up email
✓ Returns territory
✓ Filters data to territory
