Configure a Semantic Model Scheduled Refresh

Scheduled refresh automatically updates your semantic model data on a regular schedule. Configure timing, frequency, and credentials in Power BI Service to keep reports current—ensuring stakeholders always see the latest data without manual intervention.

Key Principle
Scheduled refresh keeps semantic model data synchronized with sources automatically. Balance freshness needs with resource consumption and cost.

Refresh Basics

⏰ What Is Scheduled Refresh?▼

Definition:

✓ Automatic data synchronization
✓ On predetermined schedule
✓ Power BI Service triggers

Purpose:

✓ Keep data current
✓ No manual intervention
✓ Users see latest information
🔄 Refresh vs Import▼

Import Mode:

✓ Data loaded into model
✓ Refresh updates memory
✓ Fastest queries

DirectQuery:

✓ Queries hit source
✓ No refresh needed
✓ Always current
📊 Why Schedule Refresh?▼

Benefits:

✓ Consistent timing
✓ Off-peak hours
✓ Predictable performance

Advantages:

✓ Reduces manual work

✓ Improves reliability

Credentials & Authentication

Why Needed:

✓ Service can't use your account

⚠ Wrong credentials - Use service account
⚠ No monitoring - Check history regularly
⚠ Overlapping schedules - Stagger if needed
⚠ No timeout handling - Set limits
⚠ Ignoring capacity - Monitor resource use
