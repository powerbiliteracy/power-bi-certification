Semi-Additive Measures

Semi-additive measures are additive over some dimensions (product, region) but NOT others (time). Balance sheets, inventory, and bank accounts shouldn't sum across months (take Dec 31 value only). Essential for financial reporting, balance sheet analysis, and inventory management.

Measure TypeAdditive?ExampleSolution
Fully AdditiveYes (all dims)SalesSUM()
Semi-AdditiveSome dims yesInventoryLASTNONBLANK

💡
Key Principle
Semi-additive = "Can I sum this across dimensions?" For Inventory: YES by product, region (sum all items). For Time: NO (don't sum Jan+Feb, take Feb 28 only). Solution: Use LASTNONBLANK to get latest value, ignore time dimension summation.

Understanding Semi-Additive Measures

📊 Fully Additive (Normal SUM)▼

Example: Sales

✓ Can sum by product (Product A + B = Total)
✓ Can sum by region (East + West = Total)
✓ Can sum by month (Jan + Feb + Mar = Q1)
✓ Use normal SUM()

Formula:

Sales = SUM([SalesAmount])
⚖️ Semi-Additive (Some Dimensions)▼

Example: Inventory

✓ Can sum by product (Qty A + Qty B = Total Qty)
✓ Can sum by warehouse (Warehouse 1 + 2 = Total)
❌ CANNOT sum by month (Jan + Feb ≠ March)

Why Not by Time?

• Inventory is a point-in-time snapshot
• March 31 inventory ≠ Jan 31 + Feb 28
• Must use last value of month only

Formula:

Inventory = LASTNONBLANK( DimDate[DateKey], SUM([InventoryQuantity]) )
❌ Non-Additive (No Dimensions)▼

Example: Unit Price

❌ Can't sum by product (Price A + Price B ≠ Avg price)
❌ Can't sum by month (Jan price + Feb price ≠ useful)
✓ Must AVERAGE or use custom logic

Formula:

Avg Price = AVERAGE([UnitPrice])
🎯 Common Semi-Additive Cases▼

Balance Sheet Items:

✓ Assets (equipment, cash) - use LASTNONBLANK
✓ Liabilities (debt) - use LASTNONBLANK
✓ Equity - use LASTNONBLANK

Inventory/Stock:

✓ Warehouse stock - use LASTNONBLANK
✓ Product inventory - use LASTNONBLANK

Other Point-in-Time:

✓ Account balance - use LASTNONBLANK
✓ Headcount - use LASTNONBLANK (end of period)

Real Examples

📊 Inventory Quantity (Semi-Additive)▼

The Problem:

❌ SUM([Qty]) by month = 100 + 80 + 90 = 270 (WRONG)
✓ Correct: March 31 quantity = 90

Solution:

Inventory = LASTNONBLANK( DimDate[DateKey], SUM([InventoryQty]) )

Why it works:

✓ Takes last date's value = March 31 = 90
✓ Still sums across products/warehouses
💰 Balance Sheet (Semi-Additive)▼

The Problem:

❌ Assets Jan 1: $1M, Feb 1: $1.1M, Mar 1: $1.2M
❌ SUM = $3.3M (WRONG - should be $1.2M)

Solution:

Total Assets = LASTNONBLANK( DimDate[DateKey], SUM([Assets]) )

Result:

✓ Takes March 1 value = $1.2M (correct)
👥 Headcount (Semi-Additive)▼

The Problem:

❌ Jan 31: 100 employees, Feb 28: 110, Mar 31: 120
❌ SUM = 330 (WRONG - should be 120 at period end)

Solution:

Headcount = LASTNONBLANK( DimDate[DateKey], SUM([EmployeeCount]) )

Result:

✓ Takes March 31 = 120 (period-end count)
💵 Cash Balance (Non-Additive)▼

The Problem:

❌ Can't sum by dimension (Jan $500K + Feb $600K ≠ useful)
❌ Must use last value only

Solution:

Cash Balance = LASTNONBLANK( DimDate[DateKey], SUM([CashAmount]) )

Alternative (if averaging needed):

Avg Cash = AVERAGE([CashAmount])

Key Formulas

🔧 LASTNONBLANK Formula▼

Syntax:

LASTNONBLANK(Column, Expression)

Parameters:

• Column: Date column (usually DateKey)
• Expression: What to calculate (SUM, AVG, etc)

What it does:

✓ Finds last date with non-blank value
✓ Returns that date's value only
✓ Ignores earlier dates

Example:

// For each product/region combo: // Find last date, get SUM at that date Inventory = LASTNONBLANK( DimDate[DateKey], SUM([QtyOnHand]) )
⏰ FIRSTNONBLANK Formula▼

When to use:

✓ Opening balance (first day of period)
✓ Beginning inventory

Example:

Opening Inventory = FIRSTNONBLANK( DimDate[DateKey], SUM([QtyOnHand]) )
📊 AVERAGE Formula (Non-Additive)▼

When to use:

✓ Average price (additive doesn't make sense)
✓ Average balance over period

Example:

Avg Price = AVERAGE([UnitPrice]) Avg Cash Balance = AVERAGE([CashBalance])
🎯 Custom Semi-Additive (Advanced)▼

Complex logic:

Ending Balance = CALCULATE( SUM([Balance]), FILTER( ALL(DimDate), DimDate[DateKey] = MAX(DimDate[DateKey]) ) )

Use when:

✓ LASTNONBLANK not sufficient
✓ Need conditional logic
✓ Multiple aggregations needed

Real-World Scenarios

Scenario 1: Monthly Inventory Report
Track ending inventory by month and product

Data:

FactInventory: Date, Product, Warehouse, QtyOnHand

Measure:

Inventory = LASTNONBLANK( DimDate[DateKey], SUM([QtyOnHand]) )

Result:

✓ By month: Shows month-end quantity
✓ By product: Sums all warehouses
✓ By warehouse: Single location qty
Scenario 2: Financial Balance Sheet
Assets, liabilities, equity (period-end values)

Data:

FactBalanceSheet: Date, Account, Amount

Measure:

Total Assets = LASTNONBLANK( DimDate[DateKey], SUM([Assets]) )

Use Case:

✓ Quarterly reporting (Dec 31, Mar 31, etc)
✓ Balance sheet by account
✓ Trend analysis (Q1, Q2, Q3, Q4)
Scenario 3: HR Headcount Reporting
Employee count at period end (by department)

Data:

FactHeadcount: Date, Department, EmployeeCount

Measure:

Headcount = LASTNONBLANK( DimDate[DateKey], SUM([EmployeeCount]) )

Result:

✓ Monthly headcount by dept
✓ Headcount growth tracking
Scenario 4: Opening vs Closing Balances
Show beginning and ending values

Formulas:

Opening Balance = FIRSTNONBLANK( DimDate[DateKey], SUM([Balance]) ) Closing Balance = LASTNONBLANK( DimDate[DateKey], SUM([Balance]) ) Period Change = [Closing] - [Opening]

Use Case:

✓ Bank reconciliation
✓ Monthly variance analysis

Best Practices

✓ Semi-Additive Best Practices▼
✓ Use LASTNONBLANK for point-in-time data
✓ Use FIRSTNONBLANK for opening balances
✓ Always have complete date dimension
✓ Name clearly (InventoryQty, AssetValue)
✓ Document why semi-additive (inventory is snapshot)
✓ Test across time periods (month, quarter, year)
✓ Validate against source system
⚠️ Common Mistakes▼
⚠ Using SUM() for inventory (wrong!)
⚠ Forgetting date dimension in LASTNONBLANK
⚠ Missing values in fact table (dates with no data)
⚠ Not documenting why semi-additive
⚠ Using average when last-value needed
⚠ Circular dependencies in formulas
⚠ Not testing edge cases (month boundaries)
💡 Advanced Techniques▼

1. Validate Data Completeness:

Has Inventory = IF( LASTNONBLANK(DimDate[DateKey], 1) > 0, 1, 0 )

2. Rolling Balance (Last N Days):

Last 7 Days Avg = CALCULATE( AVERAGE([Balance]), FILTER(ALL(DimDate), [DateKey] >= MAX([DateKey]) - 7) )

3. Period Comparison:

Previous Period = CALCULATE( [Inventory], DATEADD(DimDate[DateKey], -1, MONTH) )
