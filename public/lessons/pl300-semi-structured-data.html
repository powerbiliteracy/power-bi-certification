What is Semi-Structured Data?

Semi-structured data has some organization but isn't formatted in strict table rows/columns. Common formats: JSON (nested objects/arrays), XML (hierarchical tags), HTML tables (sometimes inconsistent). Power Query helps flatten this into usable tables.

Common Formats

📄 JSON (objects, arrays)
📋 XML (hierarchical)
🌐 HTML Tables
📑 Nested Records
🔗 REST API responses

Key Challenges

⚠️ Nested structures
⚠️ Variable hierarchy
⚠️ Missing fields
⚠️ Array expansion
⚠️ Type conflicts
💡
Key Principle
Power Query's "Expand" and "Extract" functions handle semi-structured data. Start simple: expand one level at a time, then flatten to your desired structure.

Flattening & Extraction Techniques

🔍 Expand Records ▼

Expand nested record fields into separate columns.

Example: Address record with Street, City, Zip
# Before: Single "Address" column with nested data # After: Separate Street, City, Zip columns Table.ExpandRecordColumn(Source, "Address", {"Street", "City", "Zip"}, {"Address.Street", "Address.City", "Address.Zip"})

When to use: Nested objects (records) with known fields

📂 Expand Lists/Arrays ▼

Expand array fields into multiple rows (one per item).

Example: Customer with multiple orders
# Before: One row per customer, Orders = [Order1, Order2, Order3] # After: Multiple rows per customer, one order per row Table.ExpandListColumn(Source, "Orders")

When to use: Arrays/lists that need to become separate rows

🔄 Extract Nested Data ▼

Navigate into nested structures and extract specific values.

Example: API response with data[0].value.name
# Drill down through nested structure Source = Json.Document(File.Contents("response.json")), Data = Source[data], FirstRecord = Data{0}, Value = FirstRecord[value], Name = Value[name] in Name

When to use: Deeply nested structures, specific path extraction

⚡ Power Query UI Expansion ▼

Use the expand icon (⇄) in Power Query Editor for semi-automatic flattening.

1
Load JSON/API data into Power Query
Source should show records/objects
2
Click expand icon (⇄) on column header
Shows available fields to expand
3
Select fields to add as columns
Repeat for each nested level needed
4
Continue until data is flat
M code is generated automatically
🎯 Handle Missing/Inconsistent Fields ▼

Semi-structured data often has optional fields that aren't always present.

Solution: Use error handling and defaults
# Safely extract field with default value try record[optional_field] otherwise "N/A" // Default value if missing

Impact: Prevents transformation failures due to missing data

Real-World Scenarios

Scenario 1: JSON from REST API
API returns nested JSON with customers and their orders

Data Structure:

{ "customers": [ { "id": 1, "name": "John", "orders": [ {"id": 101, "amount": 500}, {"id": 102, "amount": 300} ] } ] }

Transformation Steps:

1
Load JSON and get "customers" array
2
Expand customer records (id, name, orders)
3
Expand orders list into rows
4
Expand order records (id, amount)

Result: Flat table with customer_id, customer_name, order_id, order_amount

Scenario 2: XML File Structure
XML with hierarchical product categories

Data Structure:

<products> <category name="Electronics"> <product> <id>1</id> <name>Laptop</name> <price>999</price> </product> </category> </products>

Power Query Approach:

1
Load XML.Tables() to parse structure
2
Extract category and product attributes
3
Flatten into category, product, price columns

Result: Table with category, id, name, price per row

Scenario 3: Inconsistent API Response
Some records missing optional phone field

Challenge:

API sometimes returns phone, sometimes doesn't. Expansion fails on missing field.

Solution:

# Check field existence before expanding = Table.AddColumn(Source, "Phone", each if Record.HasFields([Address], "phone") then [Address][phone] else "N/A")

Result: Table with Phone column, "N/A" for missing values

Scenario 4: Deeply Nested Data
Multi-level hierarchy: company → department → team → employee

Challenge:

Data nested 4+ levels deep. Must expand one level at a time.

Strategy:

1
Expand Level 1: Company fields
2
Expand departments array into rows
3
Expand department records
4
Continue until completely flat

Result: One row per employee with company, dept, team context preserved
