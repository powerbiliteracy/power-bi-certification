Create Visual Calculations by Using DAX

DAX (Data Analysis Expressions) enables you to create calculations beyond basic summarization. Measures and calculated columns respond to filter context, enabling dynamic analysis and sophisticated business logic in your reports.

TypePurposeContext
MeasureAggregate values dynamicallyFilter aware
Calculated ColumnRow-level computationStatic per row
Quick MeasurePre-built templatesGuided creation
DAX FunctionSpecific calculationsContext dependent
💡
Key Principle
Measures change based on filters (dynamic). Calculated columns are static per row. Choose wisely based on your need.

Calculation Types

📊 Measures vs Columns▼

Measures:

✓ Dynamic, context-aware
✓ Recalculate based on filters
✓ Efficient (not stored)
✓ Used in aggregations

Calculated Columns:

✓ Static, computed once
✓ Row-level calculations
✓ Stored in model
✓ Good for categorization
⚡ When to Use Each▼

Use Measures For:

✓ Sales totals
✓ Averages by category
✓ Running totals
✓ Year-over-year comparisons

Use Columns For:

✓ Age brackets
✓ Revenue categories
✓ Status flags
✓ Concatenation

Measures

📈 Creating Measures▼

Basic Steps:

1. Right-click table → New Measure
2. Enter formula: =SUM(Sales[Amount])
3. Name and format
4. Use in visuals

Common Functions:

SUM: Total values
AVERAGE: Mean calculation
COUNT: Row count
MAX/MIN: Highest/lowest
🎯 Implicit vs Explicit▼

Implicit Measures:

✓ Auto-created by Power BI
✓ Drag numeric field to visual
✓ Default aggregation (Sum)

Explicit Measures:

✓ Manually created formulas
✓ More control and logic
✓ Reusable across report
✓ Recommended practice
🔄 Filter Context▼

What is Filter Context?

✓ Filters applied to visual
✓ Measures adapt automatically
✓ Example: Sales by Region

Example:

Total Sales = SUM(Sales[Amount])
If Region = North selected:
Formula sums only North sales

Calculated Columns

🔢 Creating Columns▼

Basic Steps:

1. Right-click table → New Column
2. Formula: =[Sales]/[Units]
3. Press Enter to compute all rows
4. Use in visuals like any field

Common Use Cases:

✓ Profit = Revenue - Cost
✓ Age = TODAY() - BirthDate
✓ Category = IF(Sales>1000, "High", "Low")
⚖️ Row Context▼

What is Row Context?

✓ Each row evaluated separately
✓ Access to current row values
✓ Static: computed once

Example:

Profit Margin = [Profit]/[Revenue]
Computed for each row
Result stored in model
📊 Performance Notes▼

Impacts:

✓ Stored in model (storage cost)
✓ Slows refresh time
✓ Use for truly needed fields

Alternative:

✓ Use measures instead when possible
✓ Measures are more efficient

DAX Syntax & Functions

💻 DAX Basics▼

Basic Syntax:

MeasureName = FUNCTION(column)
Format: Name = Expression
Functions: Aggregates, Logic, Time

Common Aggregates:

SUM(Sales[Amount]) AVERAGE(Sales[Amount]) COUNT(Sales[ID]) MAX(Sales[Date])
🔧 Key Functions▼

Aggregation:

SUM, AVERAGE, COUNT, MAX, MIN

Conditional:

IF: =IF(Sales > 1000, "High", "Low")
SWITCH: Multiple conditions

Time Intelligence:

YEAR, MONTH, DAY, TODAY
YTD, QTD, MTD (year/quarter/month-to-date)
📝 Writing Formulas▼

Tips:

✓ Use IntelliSense for hints
✓ Reference columns: [ColumnName]
✓ Reference tables: TableName[Column]
✓ Use parentheses for clarity

Example:

Revenue per Unit = DIVIDE( SUM(Sales[Revenue]), SUM(Sales[Units]), 0 )

Real-World Scenarios

Scenario 1: Sales KPI Dashboard

Requirement:

Total sales, average order value
Year-over-year growth
Dynamic by region/product

Solution:

Measure 1: Total Sales = SUM(Sales[Amount])
Measure 2: Avg Order = AVERAGE(Sales[Amount])
Measure 3: YoY Growth = (CurrentYear - PriorYear) / PriorYear
Scenario 2: Customer Segmentation

Requirement:

Categorize customers: High/Medium/Low value
Based on lifetime spending
Use in filtering and analysis

Solution:

Calculated Column: Customer Segment
Formula: =IF([TotalSpend]>50000, "High", IF([TotalSpend]>10000, "Medium", "Low"))
Use in slicers and visuals
Scenario 3: Profit Analysis

Requirement:

Profit = Revenue - Cost
Profit margin %
Track by product/region

Solution:

Measure 1: Total Profit = SUM(Sales[Revenue]) - SUM(Sales[Cost])
Measure 2: Profit Margin = Total Profit / SUM(Sales[Revenue])
Dynamic context handles filters
Scenario 4: Inventory Tracking

Requirement:

Days inventory outstanding (DIO)
Stock status flags
Reorder alerts

Solution:

Calculated Column: Stock Status
Formula: =IF([Quantity]<[ReorderPoint], "REORDER", "OK")
Measure: Avg DIO = AVERAGE([DaysInStock])

Best Practices

✓ DAX Best Practices Checklist▼
□ Use measures for aggregations (not columns)
□ Name measures descriptively
□ Use DIVIDE for safe division
□ Avoid nested IFs (use SWITCH)
□ Comment complex formulas
□ Test formulas with sample data
□ Use DISTINCT for unique counts
□ Consider performance impact
⚠️ Common Mistakes▼
⚠ Using columns instead of measures - Prefer measures
⚠ Not using DIVIDE - Risk division by zero
⚠ Mixing row and filter context - Understand both
⚠ Overly complex formulas - Break into steps
⚠ Ignoring performance - Test on large datasets
⚠ Poor naming - Use clear, descriptive names
⚠ No documentation - Comment your logic
