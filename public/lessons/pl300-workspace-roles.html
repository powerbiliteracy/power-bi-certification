Assign Workspace Roles

Workspace roles control who can access, edit, and manage content. There are four roles with different permission levels: Admin, Member, Contributor, and Viewer. Assign roles strategically to enable team collaboration while protecting sensitive content and maintaining governance.

Key Principle
Assign minimal necessary permissions. Follow principle of least privilege for security and governance.

Workspace Roles Basics

👥 What Are Workspace Roles?▼

Definition:

✓ Access control mechanism
✓ Define permissions per user
✓ Control who can do what

Levels:

✓ Admin (highest)
✓ Member
✓ Contributor
✓ Viewer (lowest)
🎯 Role Hierarchy▼

From Most to Least Powerful:

1. Admin - Full control
2. Member - Can edit and publish
3. Contributor - Can edit, not publish
4. Viewer - Read-only access

Key Difference:

✓ Member can publish apps
✓ Contributor cannot
🔐 Why Role Assignment Matters▼

Benefits:

✓ Control data access
✓ Prevent accidental changes
✓ Enforce governance

Security:

✓ Least privilege principle
✓ Protect sensitive content

Detailed Role Descriptions

👑 Admin Role▼

Permissions:

✓ Full workspace control
✓ Add/remove users
✓ Assign roles
✓ Delete workspace
✓ Edit/publish content

When to Assign:

✓ Workspace owners
✓ Governance leads
✓ Team managers
👨‍💼 Member Role▼

Permissions:

✓ Edit all content
✓ Publish reports/apps
✓ View dashboards/reports
✓ Cannot add/remove users
✓ Cannot delete workspace

When to Assign:

✓ Active report creators
✓ Team leads
✓ Content developers
👨‍💻 Contributor Role▼

Permissions:

✓ Edit reports/dashboards
✓ View all content
✓ Create content (in My Workspace)
✓ Cannot publish apps
✓ Cannot add users

When to Assign:

✓ Report analysts
✓ Data specialists
✓ Team members
👁️ Viewer Role▼

Permissions:

✓ View reports/dashboards
✓ View datasets (metadata only)
✓ View paginated reports
✓ Cannot edit content
✓ Cannot publish

When to Assign:

✓ Executives
✓ Report consumers
✓ Stakeholders

Permission Matrix

📊 Detailed Permissions by Role▼

Admin Can:

✓ Manage workspace members
✓ Edit workspace settings
✓ Delete/restore workspace
✓ Manage all content
✓ Assign/change roles

Member Can:

✓ Create/edit/delete reports
✓ Publish to workspace
✓ Publish apps
✓ Cannot manage users

Contributor Can:

✓ Edit own reports
✓ View all content
✓ Cannot publish

Viewer Can:

✓ View reports/dashboards
✓ Read-only access
🔑 Content Permissions▼

Report Access:

✓ Member/Contributor/Viewer can view
✓ Member/Contributor can edit
✓ Only Admin controls sharing

Dataset Access:

✓ Role-based access
✓ RLS applies on top
✓ Viewer sees metadata only

Assigning and Managing Roles

⚙️ How to Assign Roles▼

Steps:

1. Workspace → Access
2. Click "Add people"
3. Enter email/name
4. Select role from dropdown
5. Click Add

Notes:

✓ Can add individuals or groups
✓ Groups recommended for teams
📝 Adding Users or Groups▼

Individual Users:

✓ Email address
✓ One by one
✓ For specific people

Security Groups:

✓ Azure AD groups
✓ Bulk add
✓ Dynamic membership
✓ Simplify management
🔄 Modifying or Removing Access▼

Change Role:

✓ Click user in access list
✓ Select new role
✓ Confirm change

Remove Access:

✓ Click remove button
✓ Confirm removal
✓ User loses access immediately

⚠️ Common Role Mistakes▼
⚠ Too many Admins - Limit access
⚠ Wrong role for task - Verify role fits
⚠ No role review - Audit periodically
⚠ Forgotten access - Remove when done
