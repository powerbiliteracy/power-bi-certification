Create and Configure a Workspace

Workspaces are containers for collaboration in Power BI Service. You can create workspaces, manage member access, assign roles (Admin, Member, Contributor, Viewer), and configure settings—enabling organized, secure content sharing across teams.

RolePermission LevelCan Do
AdminFull controlManage all, delete, settings
MemberCreate/editPublish, edit, share
ContributorLimited createEdit existing only
ViewerRead-onlyView only

Key Principle
Organize work. Control access. Enable collaboration.

Workspace Basics

🏢 What Is a Workspace?▼

Definition:

✓ Container for Power BI content
✓ Organize reports, dashboards, datasets
✓ Enable team collaboration

Purpose:

✓ Structure and organize
✓ Control permissions
✓ Manage distribution
📍 My Workspace vs New Workspace▼

My Workspace:

✓ Personal space
✓ Always available
✓ Single owner

New Workspace:

✓ Team/department space
✓ Shared content
✓ Multiple owners possible
📊 Workspace Types▼

Options:

✓ Classic workspace
✓ New workspace (recommended)

Differences:

✓ New = better permissions
✓ New = modern features
✓ Classic = legacy

Access Control

🔐 Managing Access▼

How to Add Members:

1. Workspace settings
2. Access section
3. Add member email
4. Assign role

Bulk Options:

✓ Add groups
✓ Add AAD groups
👥 User vs Group Access▼

Individual Users:

✓ One person at a time
✓ Specific permission

Groups (Better):

✓ Add whole team
✓ Easy management
✓ Auto-update membership
🚫 Removing Access▼

Process:

✓ Find member in list
✓ Click X or remove
✓ Confirm removal

Impact:

✓ Immediate revocation
✓ Can't access content

Roles and Permissions

👑 Admin Role▼

Permissions:

✓ Create/edit/delete content
✓ Manage members
✓ Configure workspace
✓ Manage apps

Responsibility:

✓ Overall workspace control
✏️ Member Role▼

Permissions:

✓ Create/edit content
✓ Publish reports
✓ Share with others
✓ Cannot manage workspace

Use Case:

✓ Report creators
✓ Content developers
📝 Contributor Role▼

Permissions:

✓ Edit existing content
✓ View only
✓ Cannot create new

Use Case:

✓ Content editors
✓ Limited creators
👀 Viewer Role▼

Permissions:

✓ View content only
✓ No editing
✓ No creation

Use Case:

✓ Dashboard consumers
✓ Report readers

Real-World Scenarios

Scenario 1: New Department

Requirement:

Set up Sales Analytics team
5 report creators, 20 viewers
Secure controlled access

Solution:

Create workspace
Add AAD group (members)
Add AAD group (viewers)
Assign roles appropriately
Scenario 2: Project-Based

Requirement:

Q4 budget analysis project
Temporary team
Sunset in January

Solution:

Create project workspace
Add core team as Members
Document sunset plan
Delete after completion
Scenario 3: Role Transitions

Requirement:

Person promoted to admin
Old role no longer needed
Transfer responsibilities

Solution:

Update workspace roles
Assign admin to new person
Remove old role
Scenario 4: Security Cleanup

Requirement:

Employee left company
Remove access immediately
Audit for compliance

Solution:

Find all workspaces
Remove from members
Transfer content if needed

Best Practices

✓ Workspace Setup Checklist▼
□ Use descriptive names
□ Add description/purpose
□ Use AAD groups for access
□ Assign appropriate roles
□ Document owners/admins
□ Plan capacity needs
□ Set up apps if needed
□ Regular access audits
⚠️ Common Mistakes▼
⚠ Too many individual users - Use groups
⚠ Wrong role assignments - Audit regularly
⚠ Unclear naming - Be descriptive
⚠ Orphaned workspaces - Assign owner
⚠ No access review - Audit quarterly
⚠ Unused workspaces - Clean up
⚠ Scope creep - Keep focused
